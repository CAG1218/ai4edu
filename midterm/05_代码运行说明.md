## 五、运行入口与启动步骤

### 5.1 后端启动

```bash
cd backend

# 安装 Python 依赖
pip install -r requirements.txt

# 初始化数据库（首次运行）
python scripts/init_infra.py

# 写入种子数据（创建默认管理员账号）
python scripts/seed_data.py

# 启动后端服务（开发模式，自动重载）
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

# 后端启动后，可访问 http://localhost:8000/docs 查看 Swagger API 文档
```

> **默认管理员账号**：`admin@ai4edu.com` / `admin123`

### 5.2 前端启动

```bash
cd frontend

# 安装 Node.js 依赖
npm install

# 启动开发服务器（支持热更新）
npx vite --host 0.0.0.0 --port 5173

# 前端启动后，访问 http://localhost:5173/ 即可使用平台
```

> **注意**：前端 `.env` 文件中 `VITE_API_BASE_URL` 已配置为 `/api/v1`（走 Vite 代理），无需修改。如需直连后端，改为 `http://localhost:8000/api/v1`。

### 5.3 完整启动顺序

1. 启动 PostgreSQL：`pg_ctl -D /path/to/data start`（或 Windows 服务）
2. 启动 Redis：`redis-server`（或 Windows 服务）
3. （可选）启动 Neo4j、ClickHouse、Elasticsearch、MinIO
4. 启动后端（见 5.1）
5. 启动前端（见 5.2）
6. 访问 `http://localhost:5173/` 开始使用

## 七、常见问题与解决方案

### Q1：后端启动后访问 `/api/v1/auth/login` 返回 500

**原因**：通常是后端进程状态异常（uvicorn 自动重载不完全）。

**解决**：
```bash
# 找到占用 8000 端口的进程并终止
netstat -ano | findstr :8000
taskkill /PID <PID> /F

# 重新启动后端
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### Q2：前端登录后一直卡在转圈状态

**原因**：前端 `services/auth.ts` 中响应数据解包层级错误，导致 `access_token` 为 `undefined`。

**解决**：确保 `services/auth.ts` 中所有方法使用 `return (res as any).data ?? res`（已在当前版本修复）。

### Q3：完成 Onboarding 引导后一直卡在引导页面

**原因**：前端仅更新了 localStorage，未调用后端接口持久化 `onboarding_completed` 字段；路由守卫读取后端用户数据，两端状态不一致。

**解决**：已在 `StepGoalView.vue` 的 `handleComplete()` 中增加后端接口调用 + `fetchCurrentUser()` 刷新状态（当前版本已修复）。

### Q4：局域网其他机器无法访问平台

**原因**：前端 `.env` 中 `VITE_API_BASE_URL` 配置为 `http://localhost:8000/api/v1`，外部机器访问时指向自己的本地端口。

**解决**：改为 `VITE_API_BASE_URL=/api/v1`（走 Vite 代理），并在 Windows 防火墙中开放 5173 和 8000 端口。

### Q5：Neo4j / ClickHouse 等依赖服务未启动，是否影响核心功能？

**不影响**。平台以降级模式运行：
- 知识图谱功能暂不可用（Neo4j 未连接）
- 遥测数据存储暂不可用（ClickHouse 未连接）
- 核心功能（登录、笔记、资源上传、AI 学伴）正常可用
